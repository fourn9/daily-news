SpaceXAI、Grok Buildをオープンソース化 ─ 秘密裏のコードベースアップロード発覚後の反応、Anthropicは月$12.5億をSpaceXAIに支払い、Anthropic/Google/Reflection合計 月$23億の"敵に燃料を売る"戦略

参照元: The New Stack (Matthew Burns) / winbuzzer / Simon Willison / Slashdot

Elon MuskのSpaceXAIが7月15日、ターミナルコーディングエージェント「Grok Build」をApache 2.0でオープンソース化した。Claude Code / OpenAI Codexへの直接攻撃で、開発者向けハーネスを無料化して競合の価格防衛を強要する狙い。7月18日にThe New Stackが背景解説を公開: BloombergによればAnthropicはSpaceXAIに月$12.5億のコンピュート支払い、Anthropic/Google/Reflection AIを合計するとSpaceXAIは月$23億の推論クラウド売上を得ている。つまり"Muskは自社Claude競合を売り物にしながら、Anthropicが競う機械を貸している"構造で、Grokが勝たなくてもSpaceXAIは儲かる。オープンソース化の直接の契機は、7月12日にユーザーの@a_green_beingがGrok Build v0.2.93がSSH鍵・パスワードマネージャーDB・写真・ドキュメントなど"ホームディレクトリ全体"をSpaceXAIクラウドにアップロードしていることを発見、12GB規模の転送を実測（5.10 GiB storage channel）した事件。Muskはデータ削除を約束、7月13日以降アップロード機能停止済。ただしGCP storage plumbingは依然残存（無効化済）、公開履歴は1コミットのみで過去挙動の検証は困難。Grokの一部コードはOpenAI CodexとOpenCodeから移植されており、Simon Willison氏が指摘。Anthropic Claude Fable 5は7月20日からMax/Team Premiumで通常制限の50%枠に組み込まれる発表もあり、AIコーディングエージェント争奪戦は激化。
