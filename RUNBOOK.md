# RUNBOOK — 朝のニュースサマリー生成ルーティン用

このファイルは、毎朝の「朝のニュースサマリー」を生成するクラウドルーティンが
**サマリーを書き始める前に必ず読む**運用ガイドです。ルーティン本体（claude.ai 側）の
プロンプトから「生成前に必ず本リポジトリの `RUNBOOK.md` を読み、その重複除外ルールに従う」
と参照させてください。ルールを変えたいときは、この 1 ファイルを編集して commit するだけで済みます。

## 生成フロー（概要）

1. リポジトリはチェックアウト済み。まず**過去分の照合**（下記「重複除外ルール」1)を行う。
2. 本日（JST）の主要ニュースを収集し、下記カテゴリに整理してサマリーHTMLを生成する。
3. `YYYY-MM-DD/index.html` を追加し、トップ `index.html` の
   `<!-- INSERT-NEXT-SUMMARY-HERE -->` 直下に新しい日付の `<li>` を先頭追記する。
4. commit & push。反映URL: https://fourn9.github.io/daily-news/<DATE>/

カテゴリ（既存踏襲）: 🤖 AI / Tech ／ 💼 ビジネス / 企業 ／ 🚀 スタートアップ / 資金調達 ／
📈 マーケット / 経済 ／ 🌏 国際 / 政治。

## 重複除外ルール（最重要 / 今回の是正点）

**目的**: 過去に公開したニュースを、本日分として再び取り上げない。
「その日でないニュース」を新規のように出すのを止める。

1. **過去分を必ず先に読む**
   書き始める前に、直近の公開済みサマリー（最低でも直近 3 日、可能なら 7 日分）の
   `YYYY-MM-DD/index.html` を読み、**既出の見出し（`<h3>`）と参照元URL（`<a href>`）**を
   一覧化しておく。これと照合してから本日の記事を選ぶ。

2. **本日性で絞る**
   掲載してよいのは、**主要な新情報（発表・出来事）が本日（JST）付**のものだけ。
   前日以前に既報の出来事の焼き直しや、過去日付のまとめ記事（例:「週刊〜 ○月○日」の
   ように本日より前の日付が入った定期ダイジェスト）は**載せない**。

3. **進行中トピックは「新展開があるときだけ」**
   すでに扱った継続トピックは、**本日新たな進展がある場合のみ**掲載する。その場合は
   必ず `続報` タグを付け、リード文を「**何が新しいか**」から書き出す。
   前日と同じ事実の言い換え・要約のやり直しは載せない。

4. **同一記事の再掲・二重掲載の禁止**
   同一 URL、または実質的に同じ見出し（表記ゆれ・和英差を正規化して照合）の記事は
   再掲しない。同じ号（同じ日）の中でも、同じ出来事を複数カテゴリに二重掲載せず、
   最も適切なカテゴリ 1 箇所に集約する。

5. **迷ったら載せない**
   本日性が確認できない、既出かどうか判断がつかない記事は除外する。

### 照合の実務メモ

- 過去 `index.html` からの抽出例:
  `grep -oE '<h3[^>]*>.*</h3>' <DATE>/index.html`（見出し）、
  `grep -oE 'href="https?://[^"]+"' <DATE>/index.html`（URL）。
- URL は末尾のトラッキングパラメータ（`?utm_...` 等）を落として比較する。
- 見出しは記号・空白・和英を無視した実質一致で判定する。
